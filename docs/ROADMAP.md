# Roadmap

This document tracks implementation progress for the project.

- Phase 1: Verifier OS before the full model
  - Implemented baseline: SkillSpec DSL hooks, metamorphic and anti-metamorphic variants, oracle registry, exact anchor ledger, regression harness, compression adversary, true per-case verifier cost profile, oracle false-positive/false-negative auditor, strict exact-output gates, seven first skill families, versioned run JSON and persisted run/fault artifacts under `runs/`.
  - Next hardening: broader generated grammars, a published JSON schema document, compatibility tests for downstream phase gates and wider oracle confidence probes.
- Phase 2: compressed core instrumentation
  - Implemented baseline: PyTorch `BitLinear` sign+mask validated in `.venv` with `pip install -e .`, shared scales, zero-state lifecycle, residual synapse buffer, activation quantization logs, expert/KV/MTP/FSP trace ledger, real layer-forward events, packed int2 ternary buffers, CPU packed dispatch records, native RawKernel diagnostic dispatch records, strict C++/CUDA extension dispatch records by default, tiled/warp/WMMA kernel variants, CUDA-event autotune with layer/global/persistent shape caches, custom fast STE autograd forward, WMMA fp16/bf16 forward packed matmul from int2 decode-shared tiles, CUDA fast STE `grad_input` kernel reading packed int2 weights, WMMA fp16/bf16 `grad_input` for aligned and padded-edge training shapes, WMMA fp16/bf16->fp32 `grad_weight` + fused `grad_bias` for aligned and padded-edge shapes, CUDA-fused requantize/pack after weight updates, versioned repack, CUDA 12.8 extension runtime doctor, LLM smoke with strict extension backend and WMMA counts, and gradient-survival/parity checks into inputs and weights through STE.
  - Real-batch evidence: `tools/train_llm.py profile-batch` runs strict Cortex LLM training with optimizer/backward/requantize/P1-P10, extension-only kernel evidence, throughput, CPU/GPU/power/VRAM monitoring and CUDA memory snapshots; `tools/train_llm.py profile-matrix` repeats that proof across multiple shapes and seeds with aggregate JSON/CSV gates plus optional throughput/GPU/VRAM/power thresholds; `tools/train_llm.py profile-autosize` selects budget-fitting shapes with the full Cortex memory estimate, searches candidate gradient-accumulation values, measures a diverse adaptive candidate frontier with strict profiles across the provided seeds and at least two candidate-measurement seeds by default, rejects observed-VRAM budget overages, explores high upper-confidence measured sources, spends extra refinement seeds on uncertain decision-frontier candidates with a longer step window and repeated runtime profiles, averages repeated profiles per seed/step observation, confirms provisional finalists with uncertainty-adaptive repeated profile windows until the winner is confirmed, confirms unconfirmed upper-confidence challengers on the decision frontier with a default round budget sized to the measured frontier, spends a dedicated extra budget plus a residual-variance adaptive extension on decision-margin resolution with adaptive confirmation steps/repeats, re-evaluates that extension after each resolution round, requires a strictly positive confirmed decision margin, ranks final measured candidates by a robust lower-confidence score instead of raw mean, then runs the strict matrix on the selected measured-passing shapes.
  - Autosize refinement now spends that extra decision-frontier budget by `expected_gain_per_cost`, so the default path prefers the refinement action with the best expected improvement per measured token and persists `refinement_budget_actions` plus a capped representative `refinement_budget_candidate_actions` audit frontier with total-count, truncation metadata and per-entry `report_selection_reason`.
  - Next hardening: broaden real Transformer shape coverage and repeat the latency, VRAM, energy, throughput and GPU occupancy profile under larger candidate grids, more seeds, more adaptive rounds, more finalist-confirmation seeds, longer LLM batches, adaptive repeat counts/step multipliers driven by residual variance and a Bayesian search layer.
- Phase 3: MTP/FSP under contract
  - Implemented baseline: PyTorch MTP heads for horizons 1/2/4/8, confidence head, temporal consistency loss, future contracts, revision mechanism, block accept/reject gates, future-contract ledger, standalone MTP/FSP micro-task calibration, direct blockwise autoregressive token acceptance, speculative decoder-step cost accounting, persisted block-contract traces, output-goal contracts beyond token IDs, checkpoint-restored output-goal decisions, inference/Frontier output-goal gates and verified answers per effective cost metric.
  - Next: calibrate the standalone heads on larger held-out micro-task distributions and compare accepted-block plus output-goal contract speedups against verified answers per cost over checkpoint variants.
- Phase 4: cognitive memory and exact anchors
  - Implemented baseline: exact recent KV, compact latent old KV with Torch embeddings, Exact Anchor Ledger preservation, query-conditioned reconstruction with forced inclusion of explicitly required anchors, anchor fidelity verifier, trainable exact/latent/drop memory policy in the LLM forward, memory-policy loss, P4 supervision from decoded input anchors, short shared-weight learned-memory ablation harness, memory-augmented inference generation for recovered anchors, audited replacement of weaker base answers, persisted memory compression/reconstruction fidelity reports, strict compiled-circuit memory bindings that P8/P9/P7 must reconstruct before Frontier FastSolve reuse, and checkpoint-resume validation that reuses restored bindings.
  - Next: scale the learned memory policy benchmark to long-context held-out anchors, compare against deterministic memory-only ablations and measure multi-cycle retention of compiled skill bindings under larger competing circuit registries.
- Phase 5: latent reasoning with certificates
  - Implemented baseline: latent proof state, PyTorch certificate head, supervised certificate-head calibration on verifier micro-tasks, short certificate schema, checksum-backed verifier, random de-latentization probes, tool-backed arithmetic/anchor/exact/compiled-circuit checks, multi-step linear algebra certificates, richer visible+hidden/property code certificates, proof-carrying answer generation, calibrated uncertainty handling, token-reduction/calibration efficiency metric, inference gating that rejects tampered proof-carrying certificates, and verified compiled-circuit contracts for Frontier FastSolve/P7 repairs.
  - Next: expand tool verification to external solver hooks and broader multi-step domains, then benchmark certificate-token savings over held-out reasoning traces.
- Phase 6: causal regression attribution
  - Implemented baseline: counterfactual probes across blocks, experts, KV mode, MTP horizon, activation precision, FSP contract and routing; real `LayerForwardEvent` probes for block restoration and activation precision; measured recovery deltas, gain per cost, normalized cause estimates, targeted repair vs global retrain cost, regression clustering and a learned attribution policy memory updated from verified P7 regrowth outcomes.
  - Next: scale the learned attribution policy over long trace corpora, multi-cause regressions and multi-cycle repair history rather than only the short verified audit loop.
- Phase 7: minimal regrowth
  - Implemented baseline: executable repair action space, artifact edits for ternary blocks/signs/scale precision, force-anchor/MTP/expert/activation/certificate/verifier/micro-family patches, gain/cost simulation, protected non-regression gate, re-crystallization annealing and default cycle-report regrowth smoke generated from real regressions.
  - Full LLM integration: accepted repairs are now applied to real Transformer state through a bounded targeted gradient patch, verified by repair-loss improvement, protected-loss non-regression, parameter delta evidence, ternary requantization, learned P6 policy feedback and persisted checkpoint/report audit fields.
  - Next: calibrate repeated regrowth schedules across long multi-corpus training runs.
- Phase 8: fast normal careful routing
  - Implemented baseline: executable fast/normal/careful inference loop with difficulty signals, budget predictor, early exit, Mixture-of-Depths `BitLinear`, latent KV reconstruction, route-capped MTP self-speculation, oracle-audited fast-path scoring, strong-path certificate verification, expert traces, packed ternary kernel dispatch records, persisted inference JSON, restored Frontier FastSolve validation on checkpoint resume and a trained autoregressive answer generator accepted by the main inference engine and cycle artifacts.
  - Next: persist and select autoregressive checkpoints across runs, benchmark path choices over larger suites, calibrate early-exit confidence and connect hardware-specific ternary kernels beyond the CPU reference dispatch.
- Phase 9: sleep phase anti-collapse
  - Implemented baseline: failure replay buffer, verified synthetic pool with mandatory trust labels, real/exogenous reservoir, tool-solved examples, metamorphic/anti-metamorphic families, anti-collapse filter, rare-skill/diversity/calibration deltas, rare-skill-boosted consolidation scheduler, causal replay batches used by the full LLM Cortex loss, and held-out gated sleep-consolidation Frontier circuits that are P4-memory-retained, persisted, FastSolve-verified, restored on resume and promoted to P10 `compiled_frontier` proposals.
  - Next: persist external real/exogenous provenance adapters and measure post-sleep retention plus sleep-promoted circuit reuse over repeated cycles.
- Phase 10: recursive improvement gate
  - Implemented baseline: proposal generator, semantically compatible archive-derived child proposal evolution, parent lineage, diversity-pressure kind selection, in-memory sandbox trainer, full persistent evolutionary archive, dynamic evaluator, Pareto/protected/calibration/diversity patch acceptance gate, reward-hacking detector, cross-skill collapse detector, rollback tokens and verifier-approved proposal replay examples that feed the full LLM Cortex loss.
  - Full LLM integration: accepted proposals now become signed bounded Transformer-parameter patches with rollback token, repair-loss improvement, protected-loss non-regression, parameter delta evidence, checkpoint/report persistence, reusable `archive.json`/`rollback.json` state through `cortex_improvement_archive_dir` across independent runs, default bounded multi-generation P10 audits via `cortex_phase_improvement_generations=2`, and verified recursive-improvement artifacts that are materialized as replayable training examples with signed patch lineage and persisted through checkpoint resume.
  - Next: scale long shared-archive evolutionary runs across real corpora, larger proposal budgets and repeated wake/sleep cycles.
- Cross-phase objective and metrics
  - Implemented baseline: executable 17-term `L_total` from the plan, objective weights, effective joule model, recursive-improvement invalidity gates, all 15 absolute metrics, persisted objective reports in `summary.json`, output-goal rejection pressure inside `L_future_contract`, and bounded objective-feedback scaling for the trainable Cortex LLM confidence regularization and phase replay losses.
  - Next: calibrate term weights empirically against long real training runs and compare checkpoint selection policies.
- Plan experiments A-E
  - Implemented baseline: named experiments for verifier fault detection, compression adversary, minimal regrowth, SlowSolve to FastSolve and auto-improvement sandbox, all with persisted metrics and pass/fail criteria including calibration/collapse checks for Experiment E.
  - Next: run the experiments over larger randomized suites and compare against trained checkpoint variants.
- Frontier skill discovery
  - Implemented baseline: fragile-skill selection, adversarial frontier task generation, slow/reference solving of source regressions and frontier variants, oracle verification, invariant extraction, micro-example distillation, structured micro-circuit task features, `BitLinear` micro-circuit compilation with DSV revalidation, verified metamorphic support recompilation, blocking held-out gate before registry promotion, runtime registry of DSV/held-out-passing compiled circuits, lexicographic coverage-specialized selection through `CompiledFrontierAgent`, registry save/load via `frontier_registry.json` plus micro-model checkpoints, P4 memory binding/reconstruction gates for compiled-circuit retention, P5 compiled-circuit contract certificates with held-out lineage and memory-binding claims, P8 `UltraFastInferenceEngine` consumption of the compiled circuit before route execution, P9 sleep-consolidation circuit promotion by coherent family, and bounded LLM phase-controller discovery that persists the registry and proves compiled FastSolve usage, accepted P7 compiled repair candidates, sleep-frontier FastSolve, and P10 `compiled_frontier` proposals in `cortex_phase_report.json`.
  - Next: scale held-out frontier generalization and competing-circuit selection across larger skill suites, sleep families, longer support loops and multi-cycle retention.
- Trainable micro-checkpoint loop
  - Implemented baseline: PyTorch answer-class micro-model and character-level autoregressive decoder with trainable projection/recurrent/head layers around an instrumented `BitLinear` core, DSV-compatible agents, verifier/sleep dataset ingestion, generated-answer checks, compiled-circuit certificates, checkpoint save/load and `summary.json` autoregressive checkpoint reports.
  - Next: use the final objective as the optimization target, evaluate held-out generalization, persist checkpoint selection across runs and benchmark MTP vs next-token-only variants under low precision.
- Full LLM pretraining bridge
  - Implemented baseline: BPE tokenizer training, streaming text-shard reader, memmap causal dataset, complete causal Transformer baseline, Cortex multi-horizon Transformer, next-token plus Cortex objective terms including learned memory policy, packed int2 ternary dispatch evidence, explicit no-silent-fallback CUDA requirement, mixed precision policy, DDP environment wiring, checkpoint persistence with corpus identity, code provenance and Cortex phase state, CSV/PNG learning curves and baseline-vs-Cortex comparison report.
  - Next: run real large-corpus GPU/DDP training, validate NCCL on hardware with multiple CUDA devices, publish multi-seed/domain learning curves, scale checkpoint size, benchmark learned-memory and packed-ternary ablations, and connect recursive-improvement proposals to persisted LLM checkpoint patches.
